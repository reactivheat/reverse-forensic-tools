# Contribution Guide

## Expectations
- Every PR includes:
  - description
  - tests (or justification)
  - doc updates if behavior changes
  - security considerations

## Review Checklist
- Does this change respect path-policy and redaction mode?
- Are guardrails applied?
- Are outputs schema-compliant and versioned?
- Are errors domain-mapped?

## Branching
- Use short-lived branches.
- Prefer small, composable PRs.
